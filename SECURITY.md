# Security Policy

## Reporting a Vulnerability

We take the security of flexoki-tmux seriously. If you discover a potential security vulnerability in this codebase, we appreciate your help in disclosing it to us responsibly.

To report a security vulnerability, please use GitHub's [private vulnerability reporting](https://github.com/codedmart/flexoki-tmux/security/advisories/new). Include:

- A clear and concise description of the vulnerability
- Steps to reproduce the vulnerability
- Potential impact of the vulnerability
- Any additional information that may help us understand and address the issue

We will acknowledge receipt of your report and provide an estimated timeline for addressing the vulnerability. We kindly request that you do not disclose the vulnerability publicly until we have had sufficient time to investigate and address the issue.

## Supported Versions

We actively maintain and support the latest version of flexoki-tmux. Please keep your installation up to date with the latest release to ensure you have the most recent security fixes.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Security Best Practices

- Keep your flexoki-tmux installation up to date with the latest version.
- Regularly review and update any dependencies used by flexoki-tmux.
- Follow secure coding practices when contributing to the codebase.

## Acknowledgments

Thanks to the security researchers and community members who responsibly disclose vulnerabilities. Your efforts help us maintain the security and integrity of flexoki-tmux.
